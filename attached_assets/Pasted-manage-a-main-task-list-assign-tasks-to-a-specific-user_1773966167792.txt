Important business logic
1. Assigned date must not change

Exactly like your script.

Rule:

when task is assigned first time → save assigned_date

if task updated later but assignee same → keep old date

if assignee changes to another user → set new assigned date for new user assignment

2. Notifications only on new assignment

When:

task assigned to user first time

or reassigned to different user

Send:

email

Google Chat webhook

3. No duplicate user sheets

In web app, user sheet is just a filtered view
So no extra copying is needed.

4. Manual sync becomes repair/rebuild action

In database system, no sheet sync needed.
But you can keep an admin button:

“Rebuild dashboard cache”

“Resend failed notifications”

“Normalize statuses”

5. Status normalization

Your current script normalizes:

gs on hold → GS onhold

coe approved → CoE Approved

In backend, do this in serializer/service before save.

Suggested database model example
Django model idea
from django.db import models
from django.contrib.auth.models import User

class University(models.Model):
    name = models.CharField(max_length=255)
    country = models.CharField(max_length=100, blank=True, null=True)

class Student(models.Model):
    full_name = models.CharField(max_length=255)
    email = models.EmailField(blank=True, null=True)
    phone = models.CharField(max_length=50, blank=True, null=True)

class Application(models.Model):
    STATUS_CHOICES = [
        ("GS Rejected", "GS Rejected"),
        ("GS approved", "GS approved"),
        ("GS document pending", "GS document pending"),
        ("GS onhold", "GS onhold"),
        ("GS submitted", "GS submitted"),
        ("GS additional document request", "GS additional document request"),
        ("In Review", "In Review"),
        ("Refund Requested", "Refund Requested"),
        ("Visa Refused", "Visa Refused"),
        ("Visa Granted", "Visa Granted"),
        ("Visa Lodged", "Visa Lodged"),
        ("CoE Requested", "CoE Requested"),
        ("CoE Approved", "CoE Approved"),
    ]

    student = models.ForeignKey(Student, on_delete=models.CASCADE)
    university = models.ForeignKey(University, on_delete=models.SET_NULL, null=True, blank=True)
    assigned_to = models.ForeignKey(User, on_delete=models.SET_NULL, null=True, blank=True)
    assigned_date = models.DateField(null=True, blank=True)
    application_status = models.CharField(max_length=100, choices=STATUS_CHOICES, blank=True)
    intake = models.CharField(max_length=100, blank=True, null=True)
    course = models.CharField(max_length=255, blank=True, null=True)
    remarks = models.TextField(blank=True, null=True)
    created_by = models.ForeignKey(User, on_delete=models.SET_NULL, null=True, blank=True, related_name="created_applications")
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
